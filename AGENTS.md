# AI Coding 工作入口

本文档是本仓库面向 AI Coding 工具的工作入口，也是本地文档和协作规约的一级目录。

它只回答三类问题：

- 开始任务前必须读取哪些协作规约
- 不同任务需要按需读取哪些开发文档
- 各类文档分别承担什么职责

它不承载具体 Agent 的完整设计边界、代码实现方案、单次任务计划、临时决策或工作进度。

作为一级目录，本文档应保持短小、稳定、可扫描。除唯一 H1 标题 `# AI Coding 工作入口` 外，正文目录层级最多到二级标题（`##`），不得使用三级及以下标题。

当本文档介绍某个下级文档时，应说明该文档的作用、读取时机、不需要读取的场景，以及它不能替代哪些其他文档。

超过 500 行时，检查脚本应给出告警，提示整理入口规则。整理应优先压缩重复说明、迁移细节内容到对应下级文档、删除临时任务状态和实现流水账，不得通过删除必要入口规则规避告警。

新增或修改本文档后，必须运行：

```bash
python scripts/check-agents-md-format.py
```

检查通过后，才能提交相关文档变更。

## 协作规约

在处理以下任务前，必须先完整读取：

1. `docs/guidelines/collaboration-preferences.md`
2. `docs/guidelines/ai-coding-behavior.md`

适用任务包括：

- 调研和计划制定
- 文档新增、修改或删除
- 代码新增、修改、重构或审查
- 测试、验证和问题排查
- Git 分支、提交、合并或清理操作

当用户要求“按照协作规约给出计划”或表达同等意图时，必须先：

1. 读取上述两份规约。
2. 完成必要的只读调研。
3. 检查当前工作区、Git 分支和未提交变更。
4. 按照协作规约规定的格式给出计划。
5. 不得修改文件或执行其他持久化操作。

读取完成前，不得直接给出计划。

## 项目背景与简介

本项目是一个本地个人 Q&A 知识库 Agent Harness，目标是把用户提供的零散 Q&A 整理为可检索、可追溯、可复用的本地知识资产。

项目通过 Agent 判断和表达、工具执行实际动作，并以 SQLite 作为 Q&A 事实来源，形成“录入、保存、检索、回答、引用来源”的知识闭环。当前开发以稳定闭环和诚实拒答为优先，不把临时对话或未经工具持久化的模型输出视为长期知识。

具体 Agent 能力、架构和数据边界以 `docs/agents/local-qa-knowledge-agent.md` 为准。

## Agent 开发边界

涉及本地个人 Q&A 知识库 Agent 的新增、修改、重构或审查时，必须读取：

- `docs/agents/local-qa-knowledge-agent.md`

该文档用于定义和约束：

- Agent 角色与职责边界
- Harness 架构与代码边界
- 工具契约
- 上下文与记忆边界
- 核心业务流程
- 数据模型
- 失败模式与测试要求

该文档不得用于记录：

- 单次任务计划
- 临时实施步骤
- Git 分支安排
- 待办事项或工作进度
- 仅对当前对话有效的决策

任务计划必须按照协作规约输出，不得写入 Agent 开发边界文档。

Agent 开发上下文文档应保持可读、可维护。`docs/agents/*.md` 超过 1000 行时，格式检查脚本应给出告警，提示需要整理文档内容。整理必须在既有模板架构内完成，不得通过拆分模板章节、删除必要设计边界或转移责任来规避行数告警。

篇幅整理应优先处理重复约束、临时任务状态、实现流水账、过细测试清单，以及不再符合当前实现边界的历史描述。

如果实现前发现具体 Agent 文档与目标需求不一致，或需要调整 Agent 的角色边界、工具契约、数据模型、核心流程、外部依赖、权限规则等设计内容，必须先修改对应 Agent 文档，并将文档变更提交到本地 Git；在文档版本被锁定后，才能开始对应代码实现。

## 代码地图

在处理以下任务前，如需要快速判断代码目录、模块职责、入口文件或测试文件位置，应读取：

- `docs/architecture/codebase-map.md`

适用任务包括：

- 代码新增、修改、重构或审查
- 测试、验证和问题排查
- 涉及模块归属、文件职责或调用入口定位的调研和计划制定
- 新增、移动、删除代码文件或目录

该文档用于快速了解当前代码结构和文件职责，只描述现有代码布局，不定义 Agent 能力边界，不替代 `docs/agents/local-qa-knowledge-agent.md`。

如果本次修改新增、移动、删除代码文件或目录，或显著改变既有文件职责，必须同步更新 `docs/architecture/codebase-map.md`。

新增或修改代码地图后，必须运行：

```bash
python scripts/check-codebase-map-format.py
```

检查通过后，才能提交相关文档变更。

## Agent 文档模板

只有在以下场景中才需要读取 `docs/templates/agent-development-context.template.md`：

- 新增具体 Agent 开发上下文文档
- 修改 Agent 开发上下文文档的统一结构
- 升级 Agent 开发上下文模板

普通任务计划、已有 Agent 的常规代码修改或代码审查，不需要读取模板。

模板只规定 Agent 开发上下文文档的稳定结构，不用于记录任务计划。

新增或修改 Agent 开发上下文模板，或新增、修改具体 Agent 开发上下文文档后，必须运行 `scripts/check-agent-doc-format.py` 检查模板与具体 Agent 文档格式；检查通过并提交文档后，才能开始对应代码实现。

如果 Agent 开发上下文模板是有意升级，必须同步更新 `scripts/check-agent-doc-format.py` 中的模板 hash 常量，并在同一次文档提交中锁定新的模板版本。
