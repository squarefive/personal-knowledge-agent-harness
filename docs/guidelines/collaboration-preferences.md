# 用户协作偏好

本文档记录本仓库中的用户协作偏好。AI Coding 工具在进行本地文档修改、代码修改、计划制定和提交前，应优先遵守本文档。

## 1. 文档修改计划

涉及本地文档修改时，计划中必须明确列出准备写入或修改的具体内容，不能只描述修改思路。

计划至少应包含：

- 目标文件路径
- 准备新增、替换或删除的章节
- 拟写入的核心文本或完整片段
- 是否需要提交到本地 Git

## 2. 代码修改计划

涉及代码修改时，必须先单独列出修改文件范围。

如果计划中的某一步包含代码修改，必须先给出该步骤对应的伪代码或逻辑草稿，再开始实际修改代码。

代码修改计划至少应包含：

- 修改文件范围
- 每个文件的修改目的
- 涉及核心逻辑变化的伪代码
- 测试或验证方式

## 3. 提交偏好

如果用户要求提交，提交前必须检查实际 diff，并基于实际变更生成提交信息。

文档设计变更和对应代码实现变更应拆分提交，除非只是修正文档中的错别字、路径或示例。
